- Na população da ontologia `med_syntoms.ttl`, a primeira coluna de cada linha do dataset é usado como sujeito dos triplos adicionados. Esse campo é normalizado ao substituír todos os espaços por underscores, substituir todos os caracteres "(" por underscores e remover todos os caracteres ")". Todas as outras colunas são usadas como as propriedades de objeto para o predicado `hasSymptom`. Todas essas colunas são normalizadas ao substituir todos os espaços por underscores, se não existir um underscore antes ou depois dos mesmos, removendo os espaços caso contrário; ao substituir todos os caracteres "(" por underscores e remover todos os caracteres ")". São filtrados todos os sintomas já existentes ao executar uma query SPARQL que obtenha todos os sintomas já existentes para a doença e subtraíndo os mesmos das colunas de cada linha.
- Na população da ontologia `med_doencas.ttl`, a primeira coluna de cada linha do dataset é usado como sujeito dos triplos adicionados. Esse campo é normalizado ao substituír todos os espaços por underscores, substituir todos os caracteres "(" por underscores e remover todos os caracteres ")". A outra coluna é utilizada como propriedade de objeto para o predicado `description` e é normalizada ao remover as plicas á frente e atrás de cada linha, se existirem, e dando escape a plicas que se encontrem no meio da descrição, se existirem.
- Na população da ontologia `med_tratamentos.ttl`, a primeira coluna de cada linha do dataset é usado como sujeito dos triplos adicionados. Esse campo é normalizado ao substituír todos os espaços por underscores, substituir todos os caracteres "(" por underscores e remover todos os caracteres ")". Todas as outras colunas são usadas como sujeitos da classe `Treatment`. Todas essas colunas são normalizadas ao substituir todos os espaços por underscores (processo aplicado duas vezes para filtrar matches sobrepostas no regex utilizado), se não existir um underscore antes ou depois dos mesmos, removendo os espaços caso contrário; ao substituir todos os caracteres "(" por underscores e remover todos os caracteres ")", e finalmente transformando as strings resultantes em `PascalCase`. São filtrados todos os tratamentos já existentes ao executar uma query SPARQL que obtenha todos os tratamentos já existentes para a doença e subtraíndo os mesmos das colunas de cada linha.
- - Na população da ontologia `med_doentess.ttl`, um número inteiro monotónico formatado como `D_<padded int>` é usado como sujeito dos triplos adicionados. O valor da chave `nome` dos elementos do dataset é utilizado como propriedade de objeto para o predicado `:name`, e cada um dos sintomas no valor da chave `sintomas` é normalizado utilizando a mesma técnica utilizada para `med_symptoms.ttl`.
